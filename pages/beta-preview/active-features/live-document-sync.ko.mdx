# 자동 문서 동기화에 대해

TeamplGPT의 <b>자동 문서 동기화</b> 기능은 문서의 변경 사항을 실시간으로 감지하여 "감시"할 수 있게 합니다. 변경 사항이 감지되면 파일이 다시 임베드되며 이 파일을 사용하는 모든 작업 공간이 자동으로 업데이트됩니다.

이 기능을 통해 문서를 참조하고 그 내용이 항상 원본 소스에 맞게 업데이트되도록 할 수 있습니다.

## 문서의 범위

### Docker

- 모든 웹사이트 링크
- <b>데이터 커넥터</b>를 통해 수집된 모든 파일 _(예: Confluence, Github, YouTube)_
- 수동으로 업로드된 파일은 **동기화되지 않습니다**. 브라우저가 사용자의 컴퓨터에서 읽을 수 없기 때문입니다.

### 데스크탑

_현재 이 미리보기는 데스크탑에서 비활성화되어 있습니다_

# 사용하는 방법

## 기능 활성화

먼저, 기능 미리보기 관리 페이지에서 기능을 활성화해야 합니다.

## 파일 변경 사항 감시 방법

기능을 활성화하면 **현재 임베드된 파일**에 "눈" 아이콘이 표시됩니다. 전체 디렉터리를 감시할 수는 없습니다. 행에 이 옵션이 없으면 이 파일은 감시할 수 없습니다.

<Image
  src="/images/beta-preview/live-document-sync/watch.png"
  height={1080}
  width={1920}
  quality={100}
/>

다른 작업 공간에 동일한 파일을 추가하면 파일이 자동으로 감시되는 것을 확인할 수 있습니다. 시스템에서 문서를 완전히 삭제하면 자동으로 감시가 해제됩니다.

## 감시 중인 파일을 쉽게 관리하고 관찰하기

모든 감시 중인 파일은 **매시간** 내용이 최신인지 확인됩니다. _최신 상태가 아닌_ 파일은 마지막 _7일_ 동안 내용이 새로 고침되지 않은 파일입니다.

**향후에는 문서를 강제로 새로 고침하거나 기본 최신 상태 시간을 변경할 수 있게 될 것입니다.**

# 요약 및 주의 사항

TeamplGPT의 자동 문서 동기화를 사용하여 파일을 감시하면 활성화된 모든 작업 공간에서 해당 문서의 모든 임베딩을 주기적으로 가져와 교체합니다.

이 기능을 사용하려면 연결된 임베더를 사용해야 하므로 리소스 문제나 비용 문제로 인해 일부 파일만 감시할 수 있습니다.

현재 애플리케이션이나 Docker 컨테이너를 닫으면 백그라운드 작업자가 프로세스를 종료하면 실행되지 않으므로 감시 중인 파일이 동기화되지 않습니다.

# 문제 해결

문서 동기화 기능에 문제가 있는 경우 기능 토글을 비활성화하면 TeamplGPT를 사용하는 동안 또는 재부팅 시 백그라운드 작업자가 실행되지 않습니다.
